Self-Hosted Platform Integration

Unified API gateway with SSO, centralized monitoring, and automated health checks

Client: Private Industry: Self-Hosted Infrastructure Year: 2024

Executive Summary

Developed a unified self-hosted platform integration system that consolidates multiple services (Seafile, Jellyfin, Gitea, Prometheus, Grafana, Vaultwarden) into a single API gateway with unified authentication, service discovery, and centralized health monitoring.

The solution eliminates the need for separate logins across services, provides a unified dashboard, and automates health checks and monitoring across all integrated services.

The Problem

Managing multiple self-hosted services required:

  • Separate authentication for each service
  • Manual health monitoring across multiple endpoints
  • No centralized view of service status
  • Complex service discovery and configuration
  • Inconsistent API access patterns

This created operational overhead, security concerns with multiple authentication systems, and difficulty in maintaining visibility across the infrastructure.

The Solution

Built a FastAPI-based unified platform that provides:

  • Unified Authentication: Single sign-on (SSO) across all services using OAuth2/JWT
  • API Gateway: Single entry point for all service APIs with consistent routing
  • Service Discovery: Automatic detection and registration of services
  • Health Monitoring: Centralized health checks for all services with automated status updates
  • Unified Dashboard: Web interface showing all services with real-time status
  • Docker Integration: Easy deployment via Docker Compose

Architecture

Client Layer

Web Browser, API Clients

↓

Platform Layer

Nginx Reverse Proxy → FastAPI Application → PostgreSQL Database

↓

Service Layer

Seafile, Jellyfin, BookStack, Gitea, Prometheus, Grafana, Vaultwarden

Technical Implementation

Core Components

  • FastAPI Application: Main API gateway with authentication, service registry, and health monitoring endpoints
  • Service Clients: Modular Python clients for each integrated service (Seafile, Jellyfin, Gitea, etc.)
  • Authentication Module: JWT token generation, OAuth2 password flow, user management with bcrypt password hashing
  • Service Registry: PostgreSQL database storing service configurations, health status, and metadata
  • Health Monitoring: Automated health checks with status tracking and alerting
  • Frontend Dashboard: Web UI for accessing all services with real-time status indicators

Key Features

  • Async HTTP clients using httpx for concurrent requests
  • Service client pattern with async context managers
  • Health check endpoints with automatic status updates
  • Service discovery through database registry
  • Unified error handling and response formatting
  • Comprehensive API documentation (OpenAPI/Swagger)

Technologies Used

FastAPI Python 3.10+ PostgreSQL SQLAlchemy JWT OAuth2 Docker Docker Compose Nginx Prometheus Grafana httpx Pydantic
