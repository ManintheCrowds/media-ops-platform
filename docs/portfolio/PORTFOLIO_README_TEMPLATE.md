## Portfolio-readiness checklist

Before marking a repo as portfolio-ready:

- [ ] **Title + tagline** — Clear project name and one-line description at the top.
- [ ] **Problem / solution / impact** — Portfolio narrative present (even if brief).
- [ ] **One diagram or screenshot** — Architecture, flow, or UI so the README is scannable.
- [ ] **Quick start works** — Clone + minimal steps to run (and you’ve verified them).
- [ ] **License set** — No "[Your License Here]" or empty; use a real license or "Proprietary — All rights reserved."
